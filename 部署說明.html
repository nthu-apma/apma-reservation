A 首次部署流程
1

確認 NAS 環境

DSM → 套件中心 → 確認已安裝 Container Manager。
DSM → 控制台 → Terminal & SNMP → 勾選 Enable SSH service。

2

上傳專案資料夾到 NAS

透過 File Station 把整個 Reservation system 資料夾上傳到：

/var/services/homes/docker/
3

SSH 連線到 NAS

在 Windows 終端機輸入：

ssh admin@192.168.0.97
4

進入專案資料夾並 Build

cd "/var/services/homes/docker/Reservation system"
sudo docker compose build
首次 build 約需 10～20 分鐘，請耐心等候。
5

啟動容器

sudo docker compose up -d
6

初始化資料庫結構

sudo docker compose exec -u root app npx prisma db push
7

建立管理員帳號

sudo docker compose exec -u root app node -e "const b=require('bcryptjs');const{PrismaClient}=require('@prisma/client');const p=new PrismaClient();b.hash(process.env.INITIAL_ADMIN_PASSWORD,12).then(h=>p.user.upsert({where:{email:process.env.INITIAL_ADMIN_EMAIL},update:{},create:{email:process.env.INITIAL_ADMIN_EMAIL,name:process.env.INITIAL_ADMIN_NAME,password:h,role:'ADMIN'}})).then(u=>{console.log('Done:',u.email);return p.\$disconnect()}).catch(e=>{console.error(e);process.exit(1)})"
成功時會顯示：Done: nthu.apma@gmail.com
B 日常更新流程（程式碼有修改）
1

上傳更新的檔案

用 File Station 把修改過的檔案上傳到 NAS，覆蓋舊版本。

注意：不要把 .env 覆蓋掉，那個檔案包含 NAS 的設定，和本機開發用的不同。
2

SSH 連線並重新 Build

ssh admin@192.168.0.97

cd "/var/services/homes/docker/Reservation system"
sudo docker compose build
sudo docker compose up -d
後續 build 因為有快取，通常 2～5 分鐘就完成（套件未更動的情況下）。
情況 預估 Build 時間
只改程式碼（未動 package.json） 2～5 分鐘
新增 npm 套件（有改 package.json） 8～15 分鐘
首次 Build 10～20 分鐘
docker compose up -d（啟動容器） 約 30 秒
C 資料庫結構有變更時（schema.prisma 有改動）

當 prisma/schema.prisma 有新增欄位或資料表時，除了上傳檔案和 rebuild 之外，還需要額外執行： 

sudo docker compose exec -u root app npx prisma db push
此指令會同步資料庫結構，但不會刪除現有資料，只會新增缺少的欄位。
D 設備圖片管理

設備圖片需放在 NAS 的指定資料夾，並使用特定的 URL 格式填入管理後台。

1

上傳圖片到 NAS

用 File Station 把圖片放到以下資料夾：

/var/services/homes/docker/Reservation system/public/images/services/
支援格式：png、jpg、jpeg、webp、gif
2

設定資料夾權限（首次放入圖片時）

SSH 連進 NAS 後執行：

sudo chmod -R 755 "/var/services/homes/docker/Reservation system/public/images/services"
3

在管理後台填入圖片 URL

格式：/api/images/services/檔名.副檔名

例如圖片檔名為 tas.png，就填：

/api/images/services/tas.png
注意：開頭必須有 /，不能直接填 images/services/tas.png。
E 常用維護指令
—

查看容器運行狀態

sudo docker compose ps
—

查看應用程式 Log（除錯用）

sudo docker compose logs app --tail=50
—

重新啟動容器（不重新 Build）

sudo docker compose restart
—

停止所有容器

sudo docker compose down
